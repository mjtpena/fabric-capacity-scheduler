# Contributing

We welcome contributions! Please follow these guidelines:

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR-USERNAME/fabric-capacity-scheduler.git`
3. Create a feature branch: `git checkout -b feature/your-feature`

## Development

- Test changes in a non-production Automation Account first
- Update documentation if adding new features
- Follow PowerShell best practices and naming conventions

## Testing

Before submitting a PR:

1. Test runbooks in a dev environment
2. Verify schedules execute correctly
3. Check that capacity pauses/resumes as expected

## Submitting Changes

1. Commit with clear messages: `git commit -m "Add feature description"`
2. Push to your fork: `git push origin feature/your-feature`
3. Open a Pull Request with a description of your changes

## Issues

Found a bug? Please open an issue with:
- Description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Your environment details (Azure region, PowerShell version, etc.)

## Code Style

- Use descriptive variable names
- Include comments for complex logic
- Add proper error handling
- Include documentation comments in PowerShell functions

Thank you for contributing!
